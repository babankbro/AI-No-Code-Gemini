# M03 — การเขียน Prompt เพื่อสร้างโค้ด

> 🏠 self-study + 🛠️ ใช้ตลอดเวิร์กช็อป · สกิลหัวใจของ Vibe Coding

## ทำไม Prompt คือสกิลสำคัญที่สุด
"AI จะเก่งแค่ไหน ขึ้นอยู่กับว่าเราสั่งมันดีแค่ไหน" — เราคือผู้กำกับ Prompt คือ "บทกำกับ"
Prompt คลุมเครือ → ผลลัพธ์มั่ว · Prompt ชัด → ตรงใจ

## สูตร C-G-R-E-F (องค์ประกอบ Prompt ที่ดี)
| | คือ |
|---|-----|
| **C** Context | บริบท งานนี้เกี่ยวกับอะไร ใครใช้ |
| **G** Goal | เป้าหมาย อยากได้อะไรกันแน่ |
| **R** Requirements | ข้อกำหนด สี/ฟีเจอร์/เทคโนโลยี |
| **E** Examples | ตัวอย่าง "ให้เหมือน..." |
| **F** Format | รูปแบบผลลัพธ์ |

> 🔗 **เชื่อมสำคัญ:** [[M05 เก็บ Requirement ให้ชัด|Requirement]] ที่ดี = วัตถุดิบของ C-G-R ในสูตรนี้พอดี

## สูตรพร้อมใช้
> สร้าง **[สิ่งที่อยากได้]** สำหรับ **[ใคร/บริบท]** โดยมี **[ข้อกำหนด]** ให้ออกมาเป็น **[รูปแบบ]**

## 6 เทคนิค
1. ให้บริบทชัด 2. เริ่มกว้าง→เจาะลึก 3. ยกตัวอย่าง/อ้างอิง 4. แตกงานเป็นขั้น 5. ระบุรูปแบบผลลัพธ์ 6. ขอให้ AI อธิบายโค้ด

## แก้และต่อยอดด้วย Prompt
- เจาะจุดแก้: "ปรับให้ปุ่มอยู่กึ่งกลาง เปลี่ยนสีเป็นเขียว"
- วาง error ให้ดู: "โค้ดนี้ขึ้น error... ช่วยหาสาเหตุและแก้" ← ใช้ใน [[M12 ทดสอบ & Deploy ขึ้นออนไลน์|ขั้น Test]]
- ขออธิบายก่อนแก้ · ขอปรับปรุง (โหลดเร็วขึ้น/อ่านง่ายขึ้น)

## ข้อควรระวัง
อย่าสั่งกว้างไป · อย่าหวังถูกครั้งเดียว (วนซ้ำคือปกติ) · ตรวจผลเสมอ อย่าเชื่อโค้ด AI 100% · **ระวังข้อมูลลับ** อย่าใส่รหัสผ่าน/ข้อมูลส่วนตัวใน prompt
