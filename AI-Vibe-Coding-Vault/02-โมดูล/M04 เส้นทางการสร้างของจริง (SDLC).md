# M04 — เส้นทางการสร้างของจริง 🆕

> 🏠 self-study + 🗣️ ทวนในห้อง · นี่คือ SDLC แต่เราไม่เรียกชื่อนี้ตอนแรก

## แนวคิดหลัก
การสร้างอะไรสักอย่างมี **ลำดับธรรมชาติ** — เราไม่สอนเป็นทฤษฎี (Waterfall/Agile) แต่ให้ผู้เรียน **"เดินจริง"** ตามลำดับนี้ตลอดคอร์ส แล้วค่อยเฉลยท้าย

ดูแผนที่หลัก: [[เส้นทาง 6 ขั้น]]

```
💡 Idea → ❓ Requirement → 🎨 UX/UI → 🛠️ Build → ✅ Test → 🚀 Deploy
```

## ทำไมต้องมีขั้นตอน (สำหรับคนไม่มีพื้น)
- กระโดดไปสั่ง AI สร้างเลยโดยไม่คิด → ได้ของที่ไม่ตรงใจ ต้องรื้อบ่อย
- มี "แผนที่" → รู้ว่าตอนนี้อยู่ขั้นไหน เหลืออะไร (ช่วยกลุ่ม C ไม่หลงทาง)
- เป็น **เช็กลิสต์ก่อนสร้าง** (ช่วยกลุ่ม A ไม่ลืมเรื่องสำคัญ)

## การเฉลยท้ายโมดูล
> "ที่เราจะทำกันทั้งคอร์สเนี่ย มืออาชีพเรียกว่า **SDLC** (วงจรการพัฒนาซอฟต์แวร์) — และสไตล์ที่เราใช้ คือทำทีละนิดแล้ววนปรับกับ AI มืออาชีพเรียก **Agile** ซึ่งบังเอิญเป็นหัวใจของ [[อภิธานศัพท์#Vibe Coding|Vibe Coding]] พอดี"

→ ได้ศัพท์ติดตัวโดยไม่ต้องท่อง
