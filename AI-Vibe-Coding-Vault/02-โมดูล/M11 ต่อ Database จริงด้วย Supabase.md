# M11 — ต่อ Database จริงด้วย Supabase 🆕

> 🛠️ ลงมือทำสด · **Day 2 เช้า**

## เป้าหมาย
ทำให้ฟอร์มจอง (จาก [[M10 สร้าง Frontend ด้วย AI]]) **บันทึกข้อมูลจริง** ลง database ที่ deploy ได้

## ลำดับลงมือ
1. **warm-up:** เปิด [[M09 Database คืออะไร & ใช้ยังไง|Google Sheets]] ดูว่า "ตาราง = แถว = คอลัมน์" จริง
2. สร้างโปรเจกต์ Supabase → สร้างตารางจาก SRS:
   - ตาราง `bookings` (การจอง): ชื่อ, เบอร์, วันเวลา, จำนวนคน
   - ตาราง `customers` (ลูกค้า)
3. **ความสัมพันธ์:** การจอง 1 อัน ผูกกับลูกค้า 1 คน (Supabase จัดการเบื้องหลัง ไม่ต้องเขียน SQL)
4. สั่ง AI เชื่อมฟอร์ม frontend → Supabase (นี่คือจุดที่ [[M08 Frontend & Backend คืออะไร|backend]] โผล่จริง)
5. ทดสอบ: กดจอง → ดูข้อมูลเข้าตารางใน Supabase

## 🔗 เส้นเชื่อม
- ตารางมาจาก [[M05 เก็บ Requirement ให้ชัด|"ข้อมูลที่ต้องเก็บ" ใน SRS]]
- "ความสัมพันธ์" = แนวคิดระดับ 2 จาก [[M09 Database คืออะไร & ใช้ยังไง]] ลงจอดที่นี่
