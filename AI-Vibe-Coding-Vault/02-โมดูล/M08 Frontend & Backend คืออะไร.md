# M08 — Frontend & Backend คืออะไร 🆕

> 🏠 self-study + 🗣️ ทวนในห้อง · **แนวคิดล้วน ไม่มี hands-on แยก**

## ขอบเขต
สอนแค่ "บทบาท" พอให้เข้าใจว่าระบบทำงานยังไง — backend โผล่ขึ้นมาจริงตอน [[M11 ต่อ Database จริงด้วย Supabase|ต่อ database]] (ดู [[สิ่งที่จงใจไม่สอน]])

## สะพานเชื่อมแนวคิด: ร้านอาหาร
| ส่วน | คือ | เปรียบเหมือน |
|------|-----|-------------|
| **Frontend** | สิ่งที่ผู้ใช้เห็นและกด | **หน้าร้าน** — เมนู โต๊ะ ป้าย (ลูกค้าเห็น) |
| **Backend** | สิ่งที่ทำงานเบื้องหลัง | **หลังร้าน/ครัว** — รับออเดอร์ไปทำ |
| **Database** | ที่เก็บข้อมูล | **คลังของ** — ครัวไปหยิบของจากคลัง |

> ตัวอย่าง: ลูกค้ากดปุ่ม "จอง" (frontend) → หลังร้านรับคำสั่ง (backend) → บันทึกการจองลงคลัง ([[M09 Database คืออะไร & ใช้ยังไง|database]])

## ทำไมไม่ลงมือ backend แยก
ใน [[อภิธานศัพท์#Vibe Coding|Vibe Coding]] ตัว AI/แพลตฟอร์ม (เช่น [[M11 ต่อ Database จริงด้วย Supabase|Supabase]]) จัดการ backend ให้เยอะมาก จุดที่ backend "โผล่จริง" สำหรับผู้เรียนคือตอนต่อ database — จึงสอนรวมที่นั่น
