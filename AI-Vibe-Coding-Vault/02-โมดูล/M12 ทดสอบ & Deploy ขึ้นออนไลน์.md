# M12 — ทดสอบ & Deploy ขึ้นออนไลน์ 🆕

> 🛠️ ลงมือทำสด · **Day 2 บ่าย** (build day — โปรเจกต์ตัวเองล้วน)

## ✅ ขั้น Test — ทดสอบแบบผู้ใช้จริง (UAT ย่อ 4 ข้อ)
1. **เดินตาม flow ที่ออกแบบ** — ลองใช้จริงตั้งแต่ต้นจนจบ ตรงกับที่คิดไว้ไหม
2. **กางเช็กลิสต์ "ต้องมี" จาก [[M05 เก็บ Requirement ให้ชัด|SRS]]** — ฟีเจอร์จำเป็นครบไหม (ปิด loop requirement→test)
3. **ลองทำผิด ๆ** — กรอกผิด/เว้นว่าง/กดรัว ดูว่าระบบพังไหม (edge case เบื้องต้น)
4. **เจอ error → วาง error ให้ AI แก้** — ใช้เทคนิคจาก [[M03 การเขียน Prompt]]

> เฉลย: "นี่คือ User Acceptance Testing แบบย่อ"

## 🚀 ขั้น Deploy — ปล่อยให้คนใช้
- Deploy เว็บแอป → ได้ **URL จริง** ที่เปิดให้คนทั้งโลกเข้าได้
- จบคอร์ส = มีเว็บใช้งานได้จริง + URL เป็นของตัวเอง

## 📌 หมายเหตุ "หลังปล่อยของ" (ไม่ใช่ขั้นแยก)
สิ่งที่กลุ่ม A ควรรู้เพื่อไม่ตกใจทีหลัง:
- **ค่าใช้จ่าย** — free tier มีลิมิต ถ้าโตต้องจ่าย
- **ข้อมูลเต็ม** — database free มีโควตา
- **แก้บั๊ก** — เจอปัญหาทีหลังกลับมาสั่ง AI แก้ได้
