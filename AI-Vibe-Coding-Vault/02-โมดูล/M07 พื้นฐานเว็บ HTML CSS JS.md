# M07 — พื้นฐานเว็บ HTML/CSS/JS + Responsive

> 🏠 self-study + 🗣️ ทวนในห้อง · "อ่านออกพอ ไม่ต้องท่อง"

## ทำไมต้องรู้ (แม้มี AI)
AI สร้างโค้ดให้ได้ แต่ถ้าเราอ่านออก เราจะ **"สั่งแก้ได้ตรงจุด"** และมั่นใจในผลงานมากขึ้น
- อ่านออก: เข้าใจว่าโค้ดที่ AI สร้างทำอะไร
- ปรับเป็น: บอกแก้ได้ตรงจุด ไม่ต้องรื้อใหม่
- คุยรู้เรื่อง: ใช้คำศัพท์ถูก AI เข้าใจตรง

## 3 ภาษาหลัก (เปรียบเหมือนสร้างบ้าน) — สะพานเชื่อมแนวคิด
| ภาษา | บทบาท | เปรียบเหมือน |
|------|-------|-------------|
| **HTML** | โครงสร้าง | "โครงบ้าน" — มีหัวข้อ ย่อหน้า ปุ่ม รูป |
| **CSS** | หน้าตา | "สีและการตกแต่ง" — สี ฟอนต์ ขนาด การจัดวาง |
| **JavaScript** | ความโต้ตอบ | "ระบบไฟฟ้า" — กดปุ่มแล้วมีอะไรเกิดขึ้น |

```html
<h1>Bean There Coffee</h1>
<p>Fresh coffee, every cup</p>
<button>Order Now</button>
```

## Responsive Design
เว็บเดียวแสดงผลพอดีทุกจอ (มือถือ/แท็บเล็ต/เดสก์ท็อป) อัตโนมัติ
- Mobile-first · Flexible Layout · Media Queries (Breakpoint)
- 📌 ข่าวดี: เราแค่บอก AI ว่า "ให้ responsive" มันจัดการให้

## คำศัพท์ที่ควรรู้
Tag · Element · Class/ID · Selector · Container · Breakpoint

## หลักคิดยุค AI
ไม่ต้องท่องทุกแท็ก — ติดตรงไหนถาม AI ("อธิบายโค้ดบรรทัดนี้") อยากแก้บอกตรง ๆ ("เปลี่ยนสีหัวข้อเป็นน้ำตาลเข้ม")
