# เส้นทาง 6 ขั้น (The 6-Step Journey)

แกนเล่าเรื่องของทั้งคอร์ส = ลำดับธรรมชาติของการสร้างของจริง ใช้แทนการสอน [[M04 เส้นทางการสร้างของจริง (SDLC)|SDLC]] เป็นทฤษฎี — ผู้เรียน **"เดินจริง"** ตามขั้น แล้วค่อยเฉลยท้ายว่ามืออาชีพเรียกว่าอะไร

```
💡 Idea ──▶ ❓ Requirement ──▶ 🎨 UX/UI ──▶ 🛠️ Build ──▶ ✅ Test ──▶ 🚀 Deploy
```

| ขั้น | คือ | โมดูลที่เกี่ยว |
|------|-----|----------------|
| 💡 **Idea** | อยากได้อะไร | [[M01 เทรนด์เว็บยุค AI & Vibe Coding]] |
| ❓ **Requirement** | ถามให้ชัดว่าต้องมีอะไร | [[M05 เก็บ Requirement ให้ชัด]] |
| 🎨 **UX/UI** | วาดหน้าตา/การใช้งาน | [[M06 ออกแบบ UX-UI เบื้องต้น]] |
| 🛠️ **Build** | สั่ง AI สร้าง | [[M03 การเขียน Prompt]], [[M10 สร้าง Frontend ด้วย AI]], [[M11 ต่อ Database จริงด้วย Supabase]] |
| ✅ **Test** | ลองใช้ + แก้ | [[M12 ทดสอบ & Deploy ขึ้นออนไลน์]] |
| 🚀 **Deploy** | ปล่อยให้คนใช้ | [[M12 ทดสอบ & Deploy ขึ้นออนไลน์]] |

## หมายเหตุ
- **จบที่ Deploy** — ขั้น Maintain (ดูแลต่อ) เป็นแค่หมายเหตุสั้นท้าย Deploy ไม่ใช่ขั้นแยก
- **เฉลยท้าย:** "ที่เราทำมาทั้งหมด มืออาชีพเรียก SDLC และสไตล์ที่เราใช้ (ทำทีละนิดแล้ววนปรับกับ AI) เรียก **Agile** ซึ่งคือหัวใจของ Vibe Coding พอดี"
- ใช้เป็น **แผนที่** ที่บอกว่าตอนนี้อยู่ขั้นไหน (ลดความหลงทางของกลุ่ม C)
