# เดโมกลาง — Bean There ระบบจอง

โปรเจกต์ตัวอย่างตัวเดียวที่ผู้สอนนำสร้างตลอดคอร์ส ต่อยอดจาก "Bean There" (ร้านกาแฟ) เดิม → ยกระดับเป็น **ระบบรับจอง/สั่งล่วงหน้า**

## ทำไมเลือกโปรเจกต์นี้
- **ต่อยอดของเดิม** ที่ลงทุนไปแล้ว เห็นพัฒนาการ landing page → ระบบจริง
- **บังคับให้ครบ stack โดยธรรมชาติ:**
  - ❓ Requirement (ใครจอง / กรอกอะไร)
  - 🎨 UX (flow การจอง)
  - 🛠️ Frontend (หน้าเมนู + ฟอร์ม)
  - 🗄️ Database (เก็บการจอง)
  - ⚙️ Backend (แนวคิด "บันทึกข้อมูล")
- **กลุ่ม A เอาไปใช้กับธุรกิจตัวเองได้ทันที** — ร้านอาหาร/คลินิก/ร้านเสริมสวย = ระบบจองเหมือนกันหมด

## SRS ฉบับย่อ (ตัวอย่าง)
**ผู้ใช้:** ลูกค้า (จองโต๊ะ), เจ้าของร้าน (ดูรายการจอง)

**ฟีเจอร์ต้องมี:**
- ลูกค้าดูเมนูแนะนำได้
- ลูกค้ากรอกฟอร์มจอง (ชื่อ, เบอร์, วัน/เวลา, จำนวนคน)
- บันทึกการจองลง database
- แสดงหน้า "จองสำเร็จ"

**ฟีเจอร์น่ามี:** อีเมลยืนยัน, หน้า admin ดูการจอง

**ข้อมูลที่ต้องเก็บ →** ส่งต่อเป็นตาราง [[M11 ต่อ Database จริงด้วย Supabase|Supabase]]

| ตาราง | คอลัมน์ |
|-------|---------|
| `customers` | ชื่อ, เบอร์โทร |
| `bookings` | ลูกค้า(ความสัมพันธ์), วันเวลา, จำนวนคน, สถานะ |
