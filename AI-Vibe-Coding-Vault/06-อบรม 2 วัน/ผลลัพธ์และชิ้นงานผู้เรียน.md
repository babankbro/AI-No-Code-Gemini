# ผลลัพธ์และชิ้นงานผู้เรียน

## ผลลัพธ์ขั้นต่ำ

เมื่อจบอบรม ผู้เรียนควรมีอย่างน้อย:

- Landing page 1 หน้า
- Dashboard จาก CSV หรือ Google Sheets
- Form หรือส่วนรับข้อมูลจากผู้ใช้
- ตัวอย่างหน้าเว็บที่ดึง API เช่น ราคาทองหรืออุณหภูมิ
- ทดลองเชื่อม Supabase
- URL จาก Vercel ที่เปิดได้จริง

## ความเข้าใจที่ต้องได้

- อธิบายได้ว่า Vibe Coding คืออะไร
- เขียน prompt ให้ AI สร้างเว็บได้ชัดขึ้น
- แยกได้ว่า frontend, data, API และ database ทำหน้าที่ต่างกันอย่างไร
- รู้ workflow พื้นฐาน: แก้ใน Antigravity -> push GitHub -> deploy Vercel
- รู้ว่า Google Sheets เหมาะกับ data ง่าย ๆ และ Supabase เหมาะกับ database จริง

## เกณฑ์ตรวจชิ้นงาน

| ด้าน | ผ่านเมื่อ |
|---|---|
| Landing page | มีเนื้อหาครบ, ปุ่มหลักชัด, responsive |
| Dashboard | แสดงข้อมูลเป็น table/card/chart อย่างน้อย 1 แบบ |
| Data | มีแหล่งข้อมูลจาก CSV, Google Sheets, API หรือ Supabase |
| Deploy | มี URL เปิดได้จริง |
| การอธิบาย | ผู้เรียนบอกได้ว่า AI ช่วยอะไร และตนเองตรวจอะไร |

## แนวทางต่อยอด

- เปลี่ยน landing page เป็นเว็บธุรกิจจริง
- เพิ่ม form เก็บ lead หรือ booking
- เปลี่ยน Google Sheets เป็น Supabase เมื่อข้อมูลเริ่มจริงจัง
- เพิ่มหน้า admin dashboard
- เพิ่มระบบ login ในอนาคต
