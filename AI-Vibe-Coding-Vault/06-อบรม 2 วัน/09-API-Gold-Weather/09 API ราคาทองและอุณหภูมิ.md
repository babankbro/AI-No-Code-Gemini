# 09 API ราคาทองและอุณหภูมิ

## เป้าหมายของช่วงนี้

- อธิบาย API ให้คนไม่มีพื้นฐานเข้าใจ
- ให้ผู้เรียนเห็นว่าเว็บสามารถดึงข้อมูลสดจากระบบอื่นได้
- ใช้ตัวอย่างราคาทองและอุณหภูมิเป็น demo ที่จับต้องง่าย

## อธิบาย API แบบง่าย

API คือช่องทางให้เว็บของเราไป "ขอข้อมูล" จากระบบอื่น แล้วเอาข้อมูลที่ได้กลับมาแสดงในหน้าเว็บ

```text
เว็บของเรา
-> ขอข้อมูลจาก API
-> API ส่งข้อมูลกลับมา
-> เว็บนำข้อมูลไปแสดง
```

## Flow การสอน 60 นาที

| นาที | เนื้อหา |
|---|---|
| 0-10 | อธิบาย API ด้วยภาพร้านอาหาร/ช่องรับข้อมูล |
| 10-20 | Demo ดูข้อมูลจาก API แบบ raw |
| 20-35 | ให้ AI ช่วยเขียนโค้ดดึงข้อมูลราคาทอง |
| 35-50 | ให้ AI ช่วยเขียนโค้ดดึงข้อมูลอุณหภูมิ |
| 50-60 | เพิ่ม loading/error state และสรุปข้อควรระวัง |

## สิ่งที่ต้องสอนเพิ่ม

- Endpoint คือ URL สำหรับขอข้อมูล
- Response คือข้อมูลที่ได้กลับมา
- JSON คือรูปแบบข้อมูลที่ API ส่งกลับมาบ่อย
- Loading คือสถานะระหว่างรอข้อมูล
- Error คือกรณีดึงข้อมูลไม่ได้

## Prompt ให้ AI อธิบาย API

```text
ช่วยอธิบาย API นี้ให้คนไม่มีพื้นฐานเข้าใจ:
[ใส่ API endpoint หรือคำอธิบาย API]

ให้อธิบายว่า:
- เว็บส่งคำขอไปที่ไหน
- ได้ข้อมูลอะไรกลับมา
- ข้อมูลส่วนไหนควรนำไปแสดง
- ถ้าดึงข้อมูลไม่ได้ควรแสดงข้อความอะไร
```

## Prompt สร้างหน้าแสดงข้อมูล API

```text
สร้างหน้าเว็บที่ดึงข้อมูลจาก API นี้:
[ใส่ API]

ต้องมี:
- loading state ระหว่างดึงข้อมูล
- error state ถ้าดึงไม่ได้
- card แสดงข้อมูลสำคัญ
- ปุ่ม refresh
- ออกแบบให้ responsive

ก่อนเขียนโค้ด ช่วยอธิบายโครงสร้างข้อมูลที่ API ส่งกลับมาก่อน
```

## จุดที่ต้องย้ำ

- API บางตัวต้องใช้ key หรือ token
- ห้ามใส่ secret key ลงใน frontend ถ้าเป็นงานจริง
- ถ้า API ล่ม เว็บเราต้องแสดงข้อความที่เข้าใจได้
- ตัวอย่างนี้เน้นเข้าใจ concept ไม่ต้องลงลึก backend

## Output ก่อนจบช่วง

- หน้าเว็บที่แสดงข้อมูลจาก API หรือ prototype ที่จำลอง API
- ผู้เรียนอธิบาย flow การดึงข้อมูลได้

## ส่งต่อช่วงถัดไป

เมื่อเข้าใจการอ่านข้อมูลแล้ว จะไปต่อกับการเก็บข้อมูลจริงใน [[10 Supabase Database - ฐานข้อมูลจริง]]
