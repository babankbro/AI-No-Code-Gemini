# 07 Requirement UI UX - จากแบบฟอร์มสู่ Wireframe

## เป้าหมายของช่วงนี้

- ให้ผู้เรียนคิดก่อนสร้าง ไม่สั่ง AI แบบกว้างเกินไป
- ใช้แบบฟอร์ม requirement เพื่อกำหนดผู้ใช้ ฟีเจอร์ และข้อมูล
- วาด user flow และ wireframe สำหรับสั่ง AI ต่อ

## Flow การสอน 60 นาที

| นาที | เนื้อหา |
|---|---|
| 0-10 | อธิบายว่าทำไมต้องเก็บ requirement ก่อน |
| 10-25 | กรอกแบบฟอร์ม: ใคร ทำอะไร เพื่ออะไร |
| 25-35 | แยกต้องมี vs น่ามี |
| 35-45 | วาด user flow |
| 45-55 | วาด wireframe 1-2 หน้า |
| 55-60 | แปลง requirement + wireframe เป็น prompt |

## แบบฟอร์มสั้น

| คำถาม | ตัวอย่าง |
|---|---|
| เว็บนี้ทำเพื่อใคร | ลูกค้าร้านกาแฟ |
| ผู้ใช้ต้องทำอะไรได้ | ดูเมนูและจองโต๊ะ |
| ฟีเจอร์ที่ต้องมี | ฟอร์มจอง, บันทึกข้อมูล, หน้า success |
| ฟีเจอร์ที่น่ามี | อีเมลยืนยัน, admin dashboard |
| ข้อมูลที่ต้องเก็บ | ชื่อ, เบอร์, วันเวลา, จำนวนคน |
| หน้าเว็บที่ต้องมี | หน้าแรก, ฟอร์ม, หน้า success |

ดูฉบับเต็ม: [[Requirement Sheet เทมเพลต]]

## User Flow ตัวอย่าง

```text
เปิดหน้าเว็บ
-> ดูข้อมูล/เมนู
-> กดปุ่มจอง
-> กรอกฟอร์ม
-> กดยืนยัน
-> เห็นหน้าจองสำเร็จ
```

## Wireframe ที่ควรให้วาด

- หน้าแรกหรือ landing page
- ฟอร์มรับข้อมูล
- หน้าแสดงผลลัพธ์หรือ dashboard

## Prompt แปลง Requirement เป็นงานสร้างเว็บ

```text
จาก requirement นี้ ช่วยออกแบบ user flow และ wireframe แบบข้อความ

Requirement:
[วางคำตอบจากแบบฟอร์ม]

ขอผลลัพธ์เป็น:
1. รายการหน้าจอที่ต้องมี
2. flow การใช้งาน
3. wireframe แบบข้อความของแต่ละหน้า
4. prompt สำหรับสั่ง AI สร้าง frontend
```

## จุดที่ต้องย้ำ

- ถ้า requirement ไม่ชัด AI จะเดา
- "ต้องมี" คือสิ่งที่ต้องทำให้เสร็จก่อน
- Wireframe ไม่ต้องสวย แค่บอกโครงสร้างให้ AI เข้าใจ

## Output ก่อนจบช่วง

- Requirement สั้น 1 หน้า
- User flow
- Wireframe
- Prompt สำหรับสร้าง/แก้เว็บ
