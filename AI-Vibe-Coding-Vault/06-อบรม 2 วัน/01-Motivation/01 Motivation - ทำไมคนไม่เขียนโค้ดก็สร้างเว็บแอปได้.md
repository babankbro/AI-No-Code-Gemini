# 01 Motivation - ทำไมคนไม่เขียนโค้ดก็สร้างเว็บแอปได้

## เป้าหมายของช่วงนี้

- ทำให้ผู้เรียนเห็นว่า AI Coding ลดกำแพงการเริ่มสร้างเว็บแอป
- วางบทบาทผู้เรียนเป็น "ผู้กำกับ AI" ไม่ใช่คนพิมพ์โค้ดเอง
- เชื่อมความคาดหวังว่า 2 วันนี้ต้องจบด้วยชิ้นงานจริงและ URL จริง

## Key Message

> เราไม่ต้องเริ่มจากจำ syntax แต่ต้องเริ่มจากคิดให้ชัด สั่งให้ดี ตรวจผลให้เป็น แล้วใช้ AI ช่วยสร้างโค้ดจริง

## Flow การสอน 30 นาที

| นาที | เนื้อหา |
|---|---|
| 0-5 | เปิดด้วยปัญหาเดิม: มีไอเดียแต่เขียนเว็บไม่เป็น |
| 5-10 | โลกใหม่: AI ช่วยคิด เขียนโค้ด แก้ error และ deploy |
| 10-15 | อธิบาย [[อภิธานศัพท์#Vibe Coding|Vibe Coding]] แบบภาษาคน |
| 15-20 | โชว์ตัวอย่างชิ้นงานที่ผู้เรียนจะทำ: landing page, dashboard, API, database |
| 20-25 | เชื่อมกับ [[เส้นทาง 6 ขั้น]]: Idea -> Requirement -> UX/UI -> Build -> Test -> Deploy |
| 25-30 | ตั้งเป้าร่วมกัน: วันที่ 21 ทุกคนควรมี URL |

## ตัวอย่างที่ควรโชว์

- เว็บ landing page ของร้าน/โครงการ
- Dashboard จาก CSV
- หน้าเว็บที่ดึงราคาทองหรืออุณหภูมิ
- เว็บที่ deploy แล้วบน Vercel

## คำถามชวนคิด

- ถ้าไม่ต้องเริ่มจากเขียนโค้ดเอง คุณอยากสร้างเว็บอะไร
- เว็บนั้นช่วยใคร
- ถ้าทำให้เสร็จใน 2 วัน เวอร์ชันเล็กที่สุดควรมีอะไรบ้าง

## จุดที่ต้องย้ำ

- AI ไม่ได้แทนการคิดของเรา
- Prompt ที่กว้างเกินไปจะได้งานที่ใช้ยาก
- งานจริงต้องมีข้อมูลและต้องเปิดให้คนอื่นใช้ได้
- คอร์สนี้ไม่ใช่คอร์สจำโค้ด แต่เป็นคอร์สสร้างของจริงด้วย AI
