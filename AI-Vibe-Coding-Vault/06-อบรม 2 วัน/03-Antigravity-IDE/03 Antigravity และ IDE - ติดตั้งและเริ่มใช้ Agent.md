# 03 Antigravity และ IDE - ติดตั้งและเริ่มใช้ Agent

## เป้าหมายของช่วงนี้

- ให้ผู้เรียนติดตั้งและเปิด Antigravity ได้
- เข้าใจพื้นที่ทำงานหลัก: file explorer, editor, terminal, preview/browser, AI agent
- ทดลองสั่ง AI สร้างโปรเจกต์เล็ก ๆ และดูผลใน browser
- เห็นภาพว่า Antigravity อยู่ตรงไหนเมื่อเทียบกับ Codex App, Claude Code และเครื่องมือ AI coding อื่น ๆ

## แนวโน้มการพัฒนาเว็บแอปในยุค AI & Vibe Coding

หลักสูตรนี้ใช้แกน **Vibe Coding**: ผู้เรียนบอกสิ่งที่อยากได้เป็นภาษาคน แล้วให้ AI ช่วยสร้างโค้ดจริง โดยผู้เรียนทำหน้าที่คิด สั่ง ตรวจ และวนแก้

> เปิดด้วยคำถาม: "ใครเคยมีไอเดียเว็บ/แอป แต่ไม่รู้จะเริ่มยังไง?"

### ย้อนดูการพัฒนาเว็บแบบเดิม

- ต้องเรียนหลายภาษา: HTML, CSS, JavaScript, backend, database
- ใช้เวลานานกว่าจะได้เว็บที่เปิดใช้งานได้จริง
- เจอ error แล้วต้องไล่เองทีละจุด
- คนจำนวนมากติดที่กำแพง "มีไอเดีย แต่เขียนไม่เป็น"

### โลกเปลี่ยนเพราะ AI

| ช่วงเวลา | ภาพรวม |
|---|---|
| 2022 | ChatGPT ทำให้คนทั่วไปเริ่มใช้ AI ช่วยคิดและเขียน |
| 2023-2024 | เครื่องมืออย่าง Copilot/Cursor ทำให้ AI ช่วยเขียนโค้ดใน editor |
| 2025 | คำว่า Vibe Coding แพร่หลาย: สั่งด้วยภาษาคน ให้ AI สร้างโค้ด |
| 2025-2026 | AI Agent เริ่มลงมือทำงานเป็นก้อน: อ่านไฟล์ แก้โค้ด รันคำสั่ง ทดสอบ และสรุปผล |

### Vibe Coding ทำงานอย่างไร

```text
1. เราสั่งด้วย Prompt
        ↓
2. AI วางแผน เขียนโค้ด รัน ทดสอบ
        ↓
3. เราตรวจผลจากหน้าเว็บ/ไฟล์/error
        ↓
4. สั่งแก้ วนซ้ำจนได้งานที่ใช้ได้
```

จุดสำคัญคือ "วนซ้ำ" เป็นเรื่องปกติ ไม่ใช่ความล้มเหลว ผู้เรียนไม่ต้องสั่งครั้งเดียวแล้วถูกหมด แต่ต้องรู้ว่าจะตรวจและสั่งแก้อย่างไร

## แนะนำเมนูและการใช้งาน Antigravity เบื้องต้น

เมื่อเปิดโปรแกรม Antigravity ขึ้นมา ผู้เรียนจะพบกับหน้าจอที่ออกแบบมาเพื่อทำงานร่วมกับ AI Agent โดยแบ่งพื้นที่หลักดังนี้:

### 1. แถบเครื่องมือและเมนูหลัก (Sidebar)
- **Explorer (รูปโฟลเดอร์):** ใช้สำหรับดู สร้าง และจัดการไฟล์/โฟลเดอร์ต่าง ๆ ในโปรเจกต์
- **Search (รูปแว่นขยาย):** ใช้ค้นหาข้อความหรือโค้ดในโปรเจกต์
- **Agent Chat (รูปหุ่นยนต์/แชท):** **จุดที่สำคัญที่สุด!** หน้าต่างสำหรับคุย สั่งงาน และให้ AI Agent ช่วยเขียนโค้ดให้เรา

### 2. พื้นที่การทำงาน (Workspace Areas)
- **Editor (ตรงกลางจอ):** พื้นที่สำหรับดูและแก้ไขไฟล์ (เหมือนโปรแกรมจัดการข้อความแต่สำหรับโค้ด)
- **Terminal (ด้านล่าง):** ช่องคำสั่งสำหรับรันระบบเบื้องหลัง (AI มักจะเป็นคนรันให้เรา)
- **Browser / Preview (ด้านขวาหรือแยกหน้าต่าง):** หน้าต่างสำหรับดูผลลัพธ์หน้าเว็บที่กำลังสร้างแบบ Real-time

### 3. สเตปการเริ่มใช้งาน Agent (Workflow เบื้องต้น)
1. **เปิดหน้าต่าง Agent:** คลิกไอคอนรูปแชทที่แถบด้านซ้าย
2. **พิมพ์คำสั่ง (Prompt):** บอกสิ่งที่ต้องการสร้างด้วยภาษาคน เช่น "สร้างหน้าเว็บร้านกาแฟ..."
3. **สังเกตการทำงาน:** Agent จะคิด วางแผน และลงมือสร้าง/แก้ไขไฟล์ให้เราเห็นสด ๆ
4. **ตรวจผลลัพธ์ (Review):** ดูผลลัพธ์ที่หน้า Preview ว่าได้หน้าเว็บตามที่สั่งไหม หรือมี Error หรือเปล่า
5. **สั่งแก้ไข (Iterate):** หากอยากเปลี่ยนสี เลื่อนตำแหน่ง หรือแก้ Error ให้พิมพ์สั่งไปตรง ๆ ในแชทเดิม เพื่อให้ AI แก้ไขต่อ

## สนามเครื่องมือ AI Coding ตอนนี้

เครื่องมือในตลาดแบ่งคร่าว ๆ ได้ 4 กลุ่ม ผู้เรียนไม่จำเป็นต้องจำทุกชื่อ แต่ควรเข้าใจว่าแต่ละแบบเหมาะกับงานต่างกัน

| กลุ่มเครื่องมือ | ตัวอย่าง | เหมาะกับ | ข้อจำกัดสำหรับผู้เรียนใหม่ |
|---|---|---|---|
| Chat Assistant | Gemini, ChatGPT, Claude web | คิดไอเดีย, เขียน prompt, อธิบาย error, ช่วยออกแบบ | ไม่ได้อยู่ใน project โดยตรง ต้องคัดลอกไปมา |
| Editor Assistant | GitHub Copilot, Cursor | ช่วยเติม/แก้โค้ดใน editor | ยังต้องเข้าใจไฟล์และ workflow พอสมควร |
| Terminal-first Agent | Claude Code | สั่งงาน coding ผ่าน terminal, เหมาะกับคนที่คุ้น command line | ผู้เรียนใหม่อาจกลัว terminal และมองภาพเว็บยาก |
| Agent Workspace / Desktop App | Codex App, Antigravity | มอบหมายงานเป็นก้อน อ่านไฟล์ แก้โค้ด รัน ทดสอบ สรุปผล | ต้องเรียนรู้ workflow และการตรวจงานของ agent |

## Codex App vs Claude Code vs Antigravity

> ใช้ช่วงนี้เพื่ออธิบายว่า "เครื่องมือเหล่านี้ทำงานคล้ายกันตรงเป็น AI coding agent แต่ต่างกันที่จุดเริ่มต้นและประสบการณ์ใช้งาน"

| เครื่องมือ | จุดเริ่มต้นการทำงาน | วิธีคิดง่าย ๆ | จุดแข็ง | เหมาะกับใคร |
|---|---|---|---|---|
| **Codex App** | Workspace/app ที่ให้ agent ทำงานกับโค้ดและไฟล์ | ผู้จัดการงาน coding หลายงานในพื้นที่เดียว | เหมาะกับการให้ agent อ่าน repo, แก้ไฟล์, รันเช็ก และสรุปงาน | คนที่ต้องการ agent ช่วยทำงานเป็น task และตรวจผลก่อนนำไปใช้ |
| **Claude Code** | Terminal/command line เป็นหลัก | เพื่อนร่วมทีมสาย terminal | เด่นกับการคุยกับ codebase, แก้ปัญหาเชิงลึก, ทำงานเร็วกับคนที่คุ้น command line | developer หรือผู้เรียนที่ไม่กลัว terminal |
| **Antigravity** | IDE + Agent + Browser/Preview | ห้องทำงานสร้างเว็บที่มี AI อยู่ในตัว | เห็นไฟล์ เห็นโค้ด เห็นหน้าเว็บ และให้ agent ทดสอบในที่เดียว | ผู้เรียนใหม่ที่ต้องสร้างเว็บให้เห็นผลเร็ว |

## ทำงานเหมือนกันตรงไหน

- รับคำสั่งภาษาคน
- อ่านไฟล์ในโปรเจกต์ได้
- แก้โค้ดได้
- รันคำสั่งหรือตรวจ error ได้
- ช่วยอธิบายสิ่งที่แก้
- ต้องมีมนุษย์ตรวจผล ไม่ควรปล่อยผ่านทุกอย่าง

## ต่างกันตรงไหน

| ประเด็น | Codex App | Claude Code | Antigravity |
|---|---|---|---|
| ประสบการณ์หลัก | App/workspace สำหรับมอบหมายงานให้ agent | Terminal-first | IDE-first + browser/preview |
| เหมาะกับมือใหม่ | กลาง | ยากกว่า เพราะอยู่กับ command line | ดี เพราะเห็นไฟล์และหน้าเว็บชัด |
| เหมาะกับงานเว็บ visual | ดีถ้ามี preview/test workflow | ต้องต่อ workflow เองมากกว่า | เด่น เพราะมี editor + browser ใน flow เดียว |
| การสอนในห้อง | ต้องอธิบายแนวคิด workspace/task | ต้องสอน terminal เพิ่ม | สอนแบบเห็นหน้าจอและผลลัพธ์ได้ทันที |
| ภาพจำสำหรับผู้เรียน | ผู้จัดการ agent | AI pair programmer ใน terminal | IDE ที่มี agent ช่วยสร้างและทดสอบเว็บ |

## ทำไมหลักสูตรนี้เลือก Antigravity

หลักสูตรนี้สอนคนไม่มีพื้นฐานให้ "สร้างเว็บแอปให้เห็นผลจริง" ไม่ใช่ฝึกเป็น developer เต็มรูปแบบตั้งแต่วันแรก ดังนั้นเครื่องมือหลักควรทำให้ผู้เรียนเห็นความสัมพันธ์ระหว่าง prompt, code, terminal และหน้าเว็บได้เร็วที่สุด

### ข้อดีของ Antigravity สำหรับคอร์สนี้

- **เห็นผลเป็นหน้าเว็บทันที**: ผู้เรียนเข้าใจง่ายกว่าเห็นแต่โค้ดหรือ terminal
- **ครบ workflow ในที่เดียว**: editor, file, terminal, preview/browser และ agent อยู่ในบริบทเดียวกัน
- **เหมาะกับการสอนสด**: ผู้สอน demo แล้วผู้เรียนทำตามได้เป็นขั้นตอน
- **Agent-first**: สั่งงานเป็นก้อนได้ เช่น "สร้าง landing page", "เพิ่ม form", "แก้ responsive"
- **ตรวจงานง่ายกว่า**: ผู้เรียนดูผลจาก browser, screenshot หรือ preview ได้ ไม่ต้องอ่านโค้ดทั้งหมด
- **เชื่อมกับ Gemini ได้ดีในเชิง workflow**: ใช้ Gemini ช่วยคิด/ร่าง prompt แล้วใช้ Antigravity ลงมือสร้าง
- **ลดแรงเสียดทานของมือใหม่**: ไม่เริ่มจาก terminal เป็นศูนย์กลาง จึงเหมาะกับกลุ่มที่ไม่เคยเขียนโปรแกรม

### แต่ต้องพูดอย่างเป็นธรรม

- ถ้าเป็น developer ที่ใช้ terminal เก่งอยู่แล้ว Claude Code อาจคล่องกว่า
- ถ้าต้องจัดการหลาย task หรือทำงานกับ repo ใหญ่ Codex App อาจเหมาะในบาง workflow
- ถ้าผู้เรียนยังไม่รู้ว่าจะสร้างอะไร Gemini/ChatGPT/Claude web ยังสำคัญมากในช่วงคิดและออกแบบ
- เครื่องมือเปลี่ยนเร็ว สิ่งที่ต้องสอนจริงคือ workflow: คิดให้ชัด -> prompt -> ตรวจ -> แก้ -> deploy

```text
สร้างเว็บหน้าเดียวสำหรับร้านกาแฟเล็ก ๆ ชื่อ Bean There

```

## ปัญหาที่มักเจอ

- Login ไม่ได้
- เครื่องช้า
- เปิด folder ผิด
- Preview ไม่ขึ้น
- ผู้เรียนไม่รู้ว่าไฟล์อยู่ตรงไหน
- AI สร้างแล้วแต่ไม่รู้ว่าต้องกดดูผลที่ไหน

## วิธีช่วยผู้เรียน

- ให้เปิด workspace เดียวกันตามชื่อที่กำหนด
- ใช้ตัวอย่าง prompt เดียวกันก่อน อย่าให้แตกโจทย์เร็วเกินไป
- ให้ทุกคนเห็นเว็บใน browser ก่อนค่อยไปขั้นต่อไป
- ถ้าเครื่องติดตั้งไม่ได้ ให้จับคู่กับเพื่อนก่อน

## Output ก่อนจบช่วง
