# อภิธานศัพท์ (Glossary)

> คำกลางของคอร์ส — ใช้ให้ตรง กันสับสน (สำเนาย่อจาก `CONTEXT.md` ที่ราก)

## Vibe Coding
การบอกสิ่งที่อยากได้เป็นภาษาคนแล้วให้ AI เขียนโค้ดจริงให้ ผู้เรียนเป็น "ผู้กำกับ AI" ไม่ใช่ผู้พิมพ์โค้ด — แต่ผลลัพธ์คือโค้ดจริงที่ deploy ได้
🚫 เลี่ยง: *No-code* (ใช้ได้เฉพาะเชิงการตลาด; no-code แท้คือเครื่องมือลากวางที่ซ่อนโค้ด)

## สะพานเชื่อมแนวคิด (Concept Bridge)
อธิบายแนวคิดเทคนิคผ่านการเปรียบเทียบกับสิ่งที่คุ้นเคย (HTML=โครงบ้าน, Database=ตู้เก็บเอกสาร) เป้าหมาย "เข้าใจพอจะสั่งและตรวจงาน AI" ไม่ใช่ "ท่องจำเพื่อเขียนเอง"

## เดโมกลาง (Central Demo)
โปรเจกต์ตัวอย่างตัวเดียวที่ผู้สอนนำสร้างตลอดคอร์ส → [[เดโมกลาง — Bean There ระบบจอง]]

## โปรเจกต์ของตัวเอง (Own Project)
ไอเดียจริงที่กลุ่ม A สร้างคู่ขนานกับเดโมกลาง ทุก hands-on ลงมือกับของตัวเองด้วย
🚫 เลี่ยง: *capstone, การบ้าน* — มันคือของจริงที่เอาไปใช้ได้

## พรอมต์ grill-me (Grill-Me Prompt)
พรอมต์ที่สั่งให้ AI สัมภาษณ์ผู้เรียนทีละคำถามจนพิมพ์ "หยุด" แล้วสรุปเป็น SRS → [[พรอมต์ grill-me เก็บ requirement]]

## SRS (เอกสารสรุปความต้องการ)
ผลลัพธ์ของขั้น Requirement — สรุป ผู้ใช้ / ฟีเจอร์ต้องมี-น่ามี / ข้อมูลที่ต้องเก็บ สอนเป็น "เฉลยท้าย" (ได้เอกสารก่อน รู้ชื่อทีหลัง)

## เส้นทาง 6 ขั้น (The 6-Step Journey)
💡 Idea → ❓ Requirement → 🎨 UX/UI → 🛠️ Build → ✅ Test → 🚀 Deploy → [[เส้นทาง 6 ขั้น]]
🚫 เลี่ยง: *Waterfall, Spiral, phase* ในเนื้อหาหลัก

## Flow การใช้งาน (User Flow)
เส้นทางที่ผู้ใช้เดินผ่านหน้าจอ ตอบ "มีกี่หน้าจอ อะไรบ้าง"

## Wireframe (ภาพร่างหน้าจอ)
ภาพร่างหยาบ (กล่อง ไม่ใส่สี) → ถ่ายรูปส่งให้ AI
🚫 เลี่ยง: *mockup* (สงวนไว้เรียกภาพสวยที่ AI สร้าง)

## ทดสอบแบบผู้ใช้จริง (UAT ย่อ)
4 ข้อ: เดินตาม flow / กางเช็กลิสต์ SRS / ลองทำผิด / วาง error ให้ AI แก้
🚫 เลี่ยง: *unit test, test case, QA*
