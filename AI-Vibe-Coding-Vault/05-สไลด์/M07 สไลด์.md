# โมดูล 07
## พื้นฐาน HTML5, CSS3, JavaScript & Responsive

รู้พออ่านออก ปรับแต่งผลงาน AI ได้ · ไม่ต้องท่องจำ

---

## สิ่งที่จะได้เรียนรู้

- HTML — โครงสร้างของหน้าเว็บ
- CSS — การแต่งหน้าตาให้สวย
- JavaScript — การเพิ่มความโต้ตอบ
- Responsive Design — เว็บปรับตามทุกจอ
- คำศัพท์ที่ใช้คุยกับ AI ให้รู้เรื่อง

---

## ทำไมต้องรู้ (แม้มี AI)

AI สร้างโค้ดให้ได้ แต่ถ้าเราอ่านออก เราจะ **"สั่งแก้ได้ตรงจุด"** และมั่นใจมากขึ้น

- 👀 **อ่านออก** — เข้าใจว่าโค้ดที่ AI สร้างทำอะไร
- 🔧 **ปรับเป็น** — บอกแก้ตรงจุด ไม่ต้องรื้อใหม่
- 💬 **คุยรู้เรื่อง** — ใช้คำศัพท์ถูก AI เข้าใจตรง

---

## 3 ภาษาหลัก = สร้างบ้าน

| ภาษา | บทบาท | เปรียบเหมือน |
|------|-------|-------------|
| **HTML** | โครงสร้าง | 🏗️ โครงบ้าน |
| **CSS** | หน้าตา | 🎨 สีและการตกแต่ง |
| **JavaScript** | ความโต้ตอบ | ⚡ ระบบไฟฟ้า |

> 🎤 สะพานเชื่อมแนวคิดหลักของคอร์ส — ย้ำภาพ "บ้าน" ให้ติด

---

## HTML — โครงสร้าง

ใช้ "แท็ก" ในวงเล็บ `< >` บอกว่าแต่ละส่วนคืออะไร

```html
<h1>Bean There Coffee</h1>
<p>Fresh coffee, every cup</p>
<button>Order Now</button>
<img src="coffee.jpg">
```

> แท็กเปิด `<...>` และแท็กปิด `</...>` ครอบเนื้อหาไว้

---

## CSS — แต่งหน้าตา

เลือกส่วนที่จะแต่งด้วย "Selector" แล้วใส่คุณสมบัติ

```css
h1 {
  color: #6B4226;
  font-size: 32px;
}
button {
  background: #C8794A;
  border-radius: 8px;
}
```

---

## JavaScript — เพิ่มความโต้ตอบ

ทำให้เว็บ "มีชีวิต" ตอบสนองเมื่อผู้ใช้ทำอะไร

- 🖱️ กดปุ่ม → มีป๊อปอัป ("ขอบคุณที่สั่ง!")
- 📝 กรอกฟอร์ม → ส่งข้อมูล
- 🔄 ดึงข้อมูล → แสดงผล (โหลดเมนู/ราคาสด ๆ)

---

## Responsive Design

เว็บเดียวกันแสดงผลพอดีทุกขนาดจอโดยอัตโนมัติ — 📱 มือถือ · 💻 แท็บเล็ต · 🖥️ เดสก์ท็อป

**ทำงานยังไง:**
- Mobile-first — ออกแบบมือถือก่อน
- Flexible Layout — ใช้สัดส่วนยืดหดได้
- Media Queries — ตั้ง "จุดเปลี่ยน" (Breakpoint)

> 📌 ข่าวดี: เราแค่บอก AI ว่า "ให้ responsive" มันจัดการให้

---

## คำศัพท์ที่ควรรู้

| คำ | คือ |
|----|-----|
| Tag | คำสั่งใน `< >` |
| Element | แท็ก + เนื้อหา = 1 ชิ้น |
| Class / ID | ป้ายชื่อไว้เลือกจัดสไตล์ |
| Selector | ตัวเลือกว่าจะแต่งส่วนไหน |
| Container | กล่องที่ห่อเนื้อหา |
| Breakpoint | จุดที่เลย์เอาต์เปลี่ยนตามจอ |

---

## ในยุค AI: อ่านออก ปรับเป็น ก็พอ

- ไม่ต้องท่องทุกแท็ก
- ติดตรงไหน ถาม AI: "อธิบายโค้ดบรรทัดนี้ให้หน่อย"
- อยากแก้ บอกตรง ๆ: "เปลี่ยนสีหัวข้อเป็นน้ำตาลเข้ม"
- ค่อย ๆ สะสมไปเรื่อย ๆ

---

## สรุป & ก้าวต่อไป

- HTML = โครงสร้าง · CSS = หน้าตา · JS = ความโต้ตอบ
- Responsive = เว็บเดียวปรับพอดีทุกจอ
- อ่านออกปรับเป็นก็พอ ไม่ต้องท่อง
