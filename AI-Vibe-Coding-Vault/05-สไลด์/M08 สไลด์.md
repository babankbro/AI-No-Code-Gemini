# โมดูล 08
## Frontend & Backend คืออะไร

เข้าใจ "หน้าร้าน" กับ "หลังร้าน" — แนวคิดล้วน ไม่ต้องลงมือ

> 🎤 โมดูลสั้น แต่ปลดล็อกความเข้าใจสำคัญ — ทำไมเว็บถึง "จำข้อมูล" ได้

---

## สิ่งที่จะได้เรียนรู้

- Frontend คืออะไร (สิ่งที่ผู้ใช้เห็น)
- Backend คืออะไร (สิ่งที่ทำงานเบื้องหลัง)
- ทั้งสองทำงานร่วมกับ Database ยังไง
- ทำไมคอร์สนี้ไม่ลงมือ backend แยก

---

## สะพานเชื่อม: ร้านอาหาร

| ส่วน | คือ | เปรียบเหมือน |
|------|-----|-------------|
| **Frontend** | สิ่งที่ผู้ใช้เห็น/กด | 🏠 หน้าร้าน (เมนู โต๊ะ ป้าย) |
| **Backend** | ทำงานเบื้องหลัง | 👨‍🍳 หลังร้าน/ครัว |
| **Database** | ที่เก็บข้อมูล | 📦 คลังของ |

---

## ทำงานร่วมกันยังไง

```
ลูกค้ากดปุ่ม "จอง"   (Frontend / หน้าร้าน)
        ↓
หลังร้านรับคำสั่ง     (Backend / ครัว)
        ↓
บันทึกการจองลงคลัง   (Database / คลังของ)
        ↓
ส่งกลับ "จองสำเร็จ"   → แสดงที่ Frontend
```

> 🎤 นี่คือสิ่งที่เกิดทุกครั้งที่เรากดปุ่มบนเว็บใด ๆ

---

## ทำไมไม่ลงมือ backend แยก

- ใน **Vibe Coding** ตัว AI/แพลตฟอร์มจัดการ backend ให้เยอะมาก
- จุดที่ backend "โผล่จริง" สำหรับเรา = ตอน **ต่อ database**
- จึงสอนรวมตอน [[M11 สไลด์|ต่อ Supabase]] — ได้เห็นของจริง ไม่ใช่ทฤษฎีลอย

> ✅ เรารู้ "บทบาท" ของ backend ก็พอจะสั่งและตรวจงาน AI ได้

---

## สรุป & ก้าวต่อไป

- Frontend = หน้าร้าน · Backend = ครัว · Database = คลัง
- กดปุ่ม → ครัวรับ → หยิบ/เก็บของจากคลัง → ตอบกลับ
- เราเข้าใจบทบาทพอ ปล่อย AI ลงมือ backend
