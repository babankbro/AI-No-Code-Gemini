# โมดูล 11
## ต่อ Database จริงด้วย Supabase

🛠️ ลงมือทำสด · Day 2 เช้า · ทำให้ฟอร์มจอง "จำข้อมูล" ได้

> 🎤 นี่คือจุดที่ frontend + backend + database มาเจอกันครบ

---

## เป้าหมายโมดูลนี้

ทำให้ฟอร์มจอง (จาก [[M10 สไลด์|โมดูล 10]]) **บันทึกข้อมูลจริง** ลง database ที่ deploy ได้

```
กดยืนยันจอง → ข้อมูลเข้าตารางใน Supabase จริง
```

---

## Warm-up: Google Sheets

ก่อนเข้า Supabase — เปิด Google Sheets ดู:
- นี่คือ "ตาราง" จริง ✅
- แถว = รายการ, คอลัมน์ = ข้อมูล ✅

> 🎤 พิสูจน์ว่า metaphor "ตู้เก็บเอกสาร" จาก [[M09 สไลด์|โมดูล 09]] เป็นเรื่องจริง จับต้องได้

---

## ขั้นที่ 1: สร้างโปรเจกต์ Supabase

- สมัคร/ล็อกอิน supabase.com
- สร้างโปรเจกต์ใหม่
- เปิดหน้า **Table Editor** — หน้าตาเหมือนสเปรดชีต!

> ไม่ต้องเขียน SQL — ใช้หน้าจอคลิก ๆ

---

## ขั้นที่ 2: สร้างตารางจาก SRS

ดึง "ข้อมูลที่ต้องเก็บ" จาก SRS มาทำเป็นตาราง:

| ตาราง `customers` | ตาราง `bookings` |
|-------------------|-------------------|
| ชื่อ | ลูกค้า (เชื่อม) |
| เบอร์โทร | วันเวลา |
| | จำนวนคน |
| | สถานะ |

> 🔗 Requirement → Database เกิดขึ้นจริงตรงนี้

---

## ขั้นที่ 3: ความสัมพันธ์

เชื่อม `bookings` → `customers` (จอง 1 อัน ผูกลูกค้า 1 คน)

- Supabase มีปุ่มสร้าง relationship ให้
- เบื้องหลังคือ "foreign key" แต่เราไม่ต้องรู้ชื่อ

> 🎤 นี่คือ "ความสัมพันธ์ระดับ 2" ที่เรียนใน [[M09 สไลด์|โมดูล 09]] ลงจอดจริง

---

## ขั้นที่ 4: เชื่อม Frontend → Supabase

```
เชื่อมฟอร์มจองในเว็บของฉันเข้ากับ Supabase
เมื่อกด "ยืนยันการจอง" ให้บันทึกข้อมูลลงตาราง bookings
```

> 🎤 จุดนี้คือที่ **backend โผล่จริง** — AI เขียน code เชื่อมต่อให้ เราตรวจผล

---

## ขั้นที่ 5: ทดสอบการบันทึก

1. กรอกฟอร์มจองในเว็บ
2. กดยืนยัน
3. เปิด Supabase Table Editor → เห็นแถวใหม่เข้ามา ✅

> 🎉 ตอนนี้เว็บ "จำข้อมูล" ได้แล้ว — ไม่ใช่แค่หน้าตา

---

## สรุป & ก้าวต่อไป

- สร้างตารางจาก SRS (ไม่ต้องเขียน SQL)
- ทำความสัมพันธ์ booking → customer
- เชื่อม frontend → Supabase = backend ทำงานจริง
- ทดสอบ: กดจอง → ข้อมูลเข้าตาราง
