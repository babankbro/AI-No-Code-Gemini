# โมดูล 03
## การใช้ Prompt เพื่อสร้างโค้ดอย่างมีประสิทธิภาพ

สกิลหัวใจของ Vibe Coding · คุยกับ AI ให้ได้ดั่งใจ

---

## สิ่งที่จะได้เรียนรู้

- ทำไม Prompt คือสกิลสำคัญที่สุด
- องค์ประกอบของ Prompt ที่ดี + สูตรพร้อมใช้
- 6 เทคนิคเขียน Prompt ให้ได้ผล
- แก้และต่อยอดงานด้วย Prompt
- ข้อควรระวัง + เช็กลิสต์ก่อนส่ง

---

## ทำไม Prompt คือสกิลสำคัญที่สุด

> "AI จะเก่งแค่ไหน ก็ขึ้นอยู่กับว่าเราสั่งมันดีแค่ไหน"

- 🎬 เราคือผู้กำกับ — Prompt คือ "บทกำกับ"
- 🔄 เข้า–ออกสัมพันธ์กัน: คลุมเครือ→มั่ว · ชัด→ตรงใจ
- 💪 ฝึกได้ พัฒนาได้ — ยิ่งเขียนบ่อยยิ่งเก่ง

---

## Prompt ที่แย่ vs ที่ดี

| ❌ กว้างเกินไป | ✅ ชัดเจนเจาะจง |
|----------------|-----------------|
| "ทำเว็บให้หน่อย" | "สร้าง Landing Page ร้านกาแฟ โทนน้ำตาล มี hero + เมนู 3 อย่าง + ปุ่มสั่งซื้อ responsive" |
| AI เดาเองหมด → ต้องแก้เยอะ | AI เข้าใจตรง → ใกล้เคียงตั้งแต่รอบแรก |

---

## องค์ประกอบ 5 อย่าง — สูตร C-G-R-E-F

| | | |
|---|---|---|
| **C** | Context | บริบท — งานนี้เกี่ยวกับอะไร ใครใช้ |
| **G** | Goal | เป้าหมาย — อยากได้อะไรกันแน่ |
| **R** | Requirements | ข้อกำหนด — สี/ฟีเจอร์/เทคโนโลยี |
| **E** | Examples | ตัวอย่าง — "ให้เหมือน..." |
| **F** | Format | รูปแบบผลลัพธ์ |

> 🎤 เชื่อมล่วงหน้า: ใน [[M05 สไลด์|โมดูล 05]] จะเห็นว่า "Requirement ที่ดี = วัตถุดิบของ C-G-R พอดี"

---

## สูตร Prompt พร้อมใช้

> สร้าง **[สิ่งที่อยากได้]**
> สำหรับ **[ใคร/บริบท]**
> โดยมี **[ข้อกำหนด: สี/ฟีเจอร์/เทคโนโลยี]**
> ให้ออกมาเป็น **[รูปแบบผลลัพธ์]**

---

## 6 เทคนิคเขียน Prompt ให้เก่ง

1. **ให้บริบทชัด** — บอกใคร อะไร เพื่ออะไร
2. **เริ่มกว้าง → เจาะลึก** — ทำทีละรอบ
3. **ยกตัวอย่าง/อ้างอิง** — "ให้เหมือนเว็บ..."
4. **แตกงานเป็นขั้น** — ทำทีละฟีเจอร์ ไม่รวบ
5. **ระบุรูปแบบผลลัพธ์** — ภาษา/โครงสร้าง/ความยาว
6. **ขอให้อธิบาย** — ให้ AI อธิบายโค้ดที่สร้าง

---

## ตัวอย่างจริง: Landing Page

```
สร้าง Landing Page ร้านกาแฟ "Bean There"
• โทนสีน้ำตาล–ครีมอบอุ่น
• Hero section พร้อมปุ่ม "สั่งเลย"
• เมนูแนะนำ 3 รายการ พร้อมรูปและราคา
• ส่วน About และ Footer พร้อมที่อยู่
• Responsive รองรับมือถือ
• ใช้ HTML, CSS, JavaScript
```

→ ครบ C-G-R-E-F: Context=ร้าน Bean There · Goal=Landing Page · Requirements=สี/ฟีเจอร์/responsive · Format=HTML/CSS/JS

---

## แก้และต่อยอดด้วย Prompt

- 🎯 **เจาะจุดแก้:** "ปรับให้ปุ่มอยู่กึ่งกลาง เปลี่ยนสีเป็นเขียว"
- 🐛 **วาง error ให้ดู:** "โค้ดนี้ขึ้น error... ช่วยหาสาเหตุและแก้"
- 📖 **ขออธิบายก่อนแก้:** "อธิบายว่าโค้ดส่วนนี้ทำอะไร แล้วค่อยปรับ"
- ⚡ **ขอปรับปรุง:** "ทำให้โหลดเร็วขึ้นและอ่านง่ายขึ้น"

---

## ข้อควรระวัง

- ❌ อย่าสั่งกว้างเกินไป — ยิ่งคลุมเครือยิ่งเดามั่ว
- ❌ อย่าหวังถูกครั้งเดียว — Vibe Coding คือทำซ้ำและปรับ
- ✅ ตรวจผลลัพธ์เสมอ — อย่าเชื่อโค้ด AI 100% ลองรันจริง
- 🔒 **ระวังข้อมูลลับ** — อย่าใส่รหัสผ่าน/ข้อมูลส่วนตัวใน Prompt

---

## เช็กลิสต์ก่อนส่ง Prompt

- [ ] บอกบริบทครบไหม — งานนี้คืออะไร ใครใช้
- [ ] ระบุข้อกำหนดชัดไหม — สี/ฟีเจอร์/เทคโนโลยี
- [ ] บอกรูปแบบผลลัพธ์ไหม
- [ ] งานใหญ่ — แตกเป็นขั้นย่อยหรือยัง
- [ ] ไม่มีข้อมูลลับหลุดเข้าไป

---

## สรุป & ก้าวต่อไป

- Prompt = สกิลสำคัญที่สุด — สั่งชัด ผลลัพธ์ตรงใจ
- จำสูตร **C-G-R-E-F** และ 6 เทคนิค
- งานไม่ตรง = คุยต่อและปรับ ไม่ใช่ยอมแพ้
