# โมดูล 12
## ทดสอบ & Deploy ขึ้นออนไลน์

🛠️ ลงมือทำสด · Day 2 บ่าย · ปล่อยของจริง + ได้ URL ของตัวเอง

> 🎤 build day! ครึ่งวันนี้เน้นโปรเจกต์ตัวเองล้วน ผู้สอนช่วยรายคน

---

## สิ่งที่จะได้ทำ

- ✅ ทดสอบแบบผู้ใช้จริง (UAT ย่อ)
- 🚀 Deploy เว็บขึ้นออนไลน์
- 🔗 ได้ URL จริงที่แชร์ได้
- 📌 รู้ว่า "หลังปล่อยของ" ต้องดูแลอะไร

---

## ✅ ขั้น Test — ทดสอบแบบผู้ใช้จริง

4 อย่างเบา ๆ:

1. 🚶 **เดินตาม flow** ที่ออกแบบ — ตั้งแต่ต้นจนจบ
2. 📋 **กางเช็กลิสต์ "ต้องมี" จาก SRS** — ครบไหม
3. 💥 **ลองทำผิด ๆ** — กรอกผิด/เว้นว่าง/กดรัว ระบบพังไหม
4. 🐛 **เจอ error → วางให้ AI แก้**

---

## 🔗 ปิด loop: Requirement → Test

> เช็กลิสต์ "ต้องมี" ที่เขียนใน [[M05 สไลด์|โมดูล 05]] = เช็กลิสต์ทดสอบวันนี้!

- ดูเมนูได้ ✅
- กรอกฟอร์มจองได้ ✅
- บันทึกการจอง ✅
- หน้า "จองสำเร็จ" ✅

> 🎤 เฉลย: นี่คือ "User Acceptance Testing" (UAT) แบบย่อ

---

## 🚀 ขั้น Deploy — ปล่อยให้คนใช้

- สั่ง AI/แพลตฟอร์ม deploy เว็บแอป
- ได้ **URL จริง** ที่เปิดให้คนทั้งโลกเข้าได้
- แชร์ลิงก์ให้เพื่อน/ลูกค้าทดลองได้ทันที

> 🎉 จบคอร์ส = มีเว็บใช้งานได้จริง + URL เป็นของตัวเอง

---

## 📌 หลังปล่อยของ (ไม่ใช่ขั้นแยก)

สิ่งที่ควรรู้เพื่อไม่ตกใจทีหลัง:

- 💰 **ค่าใช้จ่าย** — free tier มีลิมิต ถ้าโตต้องจ่าย
- 🗄️ **ข้อมูลเต็ม** — database free มีโควตา
- 🐛 **แก้บั๊ก** — เจอปัญหากลับมาสั่ง AI แก้ได้

> 🎤 เชื่อมกลับ [[M04 สไลด์|"Maintain"]] ที่เกริ่นไว้ตอนต้นคอร์ส

---

## 🎓 จบหลักสูตร!

เราเดินครบ **เส้นทาง 6 ขั้น**:

💡 Idea → ❓ Requirement → 🎨 UX/UI → 🛠️ Build → ✅ Test → 🚀 Deploy

และได้ของจริง: **เว็บแอป + URL ของตัวเอง** 🌍

> 🎤 ให้ผู้เรียนแชร์ URL กัน — celebrate!

---

## ก้าวต่อไปหลังคอร์ส

- ต่อยอดโปรเจกต์ตัวเองให้สมบูรณ์ขึ้น
- ลองฟีเจอร์ "น่ามี" ที่พักไว้
- ฝึก prompt ให้คมขึ้นเรื่อย ๆ
- ทักษะ Vibe Coding ใช้สร้างอะไรก็ได้ ไม่ใช่แค่ระบบจอง
