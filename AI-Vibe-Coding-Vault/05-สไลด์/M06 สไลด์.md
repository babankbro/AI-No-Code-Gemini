# โมดูล 06
## ออกแบบ UX/UI เบื้องต้น

วาดก่อนสร้าง — ออกแบบ "การใช้งาน" ให้ AI สร้างตาม · ขั้น 🎨 UX/UI

> 🎤 session ลงมือ ~1 ชม. เตรียมกระดาษ+ปากกาให้ผู้เรียน

---

## สิ่งที่จะได้เรียนรู้

- ความต่างของ UX (การใช้งาน) กับ UI (ความสวย)
- วาด Flow การใช้งาน
- วาด Wireframe หยาบ ๆ
- 4 หลัก usability
- แปลงภาพร่าง → prompt ให้ AI

---

## ขอบเขตโมดูลนี้

- ✅ **เราสอน: โครงสร้างการใช้งาน** (Flow + Wireframe)
- ❌ **ไม่สอน: ความสวย** (สี/ฟอนต์) → ปล่อยให้ AI + Gemini

> 🎤 ปลดล็อกความกังวล: "ไม่ต้องเป็นนักออกแบบ ไม่ต้องเก่งสี — แค่คิดให้ออกว่าผู้ใช้ทำอะไรบ้าง"

---

## UX vs UI (เข้าใจง่าย ๆ)

| UX (ประสบการณ์) | UI (หน้าตา) |
|-----------------|-------------|
| ใช้งานยังไง ลื่นไหมไหม | สวยไหม สีอะไร |
| ลำดับขั้นตอน | ปุ่ม ฟอนต์ ภาพ |
| **โมดูลนี้เน้นตรงนี้** | ปล่อย AI+Gemini |

---

## ชิ้นงาน 1 ชม. (3 ส่วน)

1. 🗺️ **Flow การใช้งาน** (~20 นาที)
2. ✏️ **Wireframe หยาบ** 2–3 หน้า (~25 นาที)
3. 🤖 **แปลงเป็น Prompt** (~15 นาที)

---

## ส่วน 1: Flow การใช้งาน

เส้นทางที่ผู้ใช้เดินผ่านหน้าจอ:

```
เปิดหน้าเมนู → เลือกวัน/เวลา → กรอกชื่อ-เบอร์
   → กดยืนยัน → เห็นหน้า "จองสำเร็จ"
```

> ตอบคำถาม: "ระบบมีกี่หน้าจอ อะไรบ้าง"

---

## ส่วน 2: Wireframe หยาบ

วาดกล่อง ๆ บนกระดาษ ว่าแต่ละหน้ามีอะไรวางตรงไหน (ยังไม่ใส่สี)

```
┌─────────────────┐
│   [โลโก้ร้าน]    │
│  รูปเมนูแนะนำ    │
│ ┌───┐┌───┐┌───┐ │
│ │ ☕││ ☕││ ☕│ │
│ └───┘└───┘└───┘ │
│  [ ปุ่มจองเลย ]  │
└─────────────────┘
```

→ **ถ่ายรูป → ส่งให้ AI อ่าน** (หรือให้ Gemini generate mockup สวยจากภาพร่าง)

---

## ส่วน 3: แปลงเป็น Prompt

บรรยาย wireframe ให้ AI:

```
สร้างหน้าจองร้านกาแฟตามภาพร่างนี้:
- ด้านบนมีโลโก้
- ตรงกลางโชว์เมนูแนะนำ 3 อย่างเรียงแนวนอน
- ด้านล่างมีปุ่ม "จองเลย" เด่น ๆ
- responsive
```

> 🔗 เชื่อมกลับ [[M03 สไลด์|Vibe Coding]] — ภาพร่างคือวัตถุดิบของ prompt

---

## 4 หลัก Usability

1. 🪜 **ขั้นตอนน้อยที่สุด** — ยิ่งคลิกน้อยยิ่งดี
2. 👁️ **ปุ่มสำคัญต้องเด่น** — หาเจอทันที
3. 💬 **บอกผู้ใช้เสมอว่าเกิดอะไรขึ้น** — กดแล้วมีตอบสนอง
4. 🧠 **อย่าให้ผู้ใช้ต้องคิด** — ป้ายชื่อชัด ไม่กำกวม

> 🎤 เฉลย: นี่คือหลัก "usability" ที่ UX designer ใช้จริง

---

## สรุป & ก้าวต่อไป

- ออกแบบ "การใช้งาน" ก่อน "ความสวย"
- Flow + Wireframe (กระดาษ→ถ่ายรูป→AI) = สิ่งที่ UX designer ทำ
- ภาพร่าง = input ของ [[M10 สไลด์|การสร้าง frontend]]
