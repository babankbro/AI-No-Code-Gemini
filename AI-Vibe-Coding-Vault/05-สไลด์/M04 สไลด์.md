# โมดูล 04
## เส้นทางการสร้างของจริง

จากไอเดียในหัว → ของจริงที่คนใช้ได้ · (มืออาชีพเรียกสิ่งนี้ว่า SDLC — แต่เก็บไว้เฉลยท้าย)

> 🎤 อย่าเพิ่งพูดคำว่า SDLC! เริ่มจากเรื่องที่ทุกคนเข้าใจ: "การสร้างอะไรก็มีลำดับของมัน"

---

## สิ่งที่จะได้เรียนรู้

- ทำไม "กระโดดไปสั่ง AI เลย" มักได้ของไม่ตรงใจ
- เส้นทาง 6 ขั้นของการสร้างของจริง
- เราจะเดินตามเส้นทางนี้ทั้งคอร์ส
- (เฉลยท้าย) สิ่งนี้คือ SDLC และ Agile

---

## ปัญหา: กระโดดข้ามขั้น

🙅 สั่ง AI ว่า "สร้างระบบจองให้หน่อย" เลย →
- AI เดาเองว่าใครใช้ ต้องเก็บอะไร หน้าตาแบบไหน
- ได้ของไม่ตรงใจ ต้องรื้อซ้ำหลายรอบ

✅ มีขั้นตอน → คิดให้ครบก่อน แล้วค่อยสั่ง → ตรงใจตั้งแต่ต้น

> 🎤 เปรียบกับสร้างบ้าน: ไม่มีใครเทปูนก่อนเขียนแบบ

---

## เส้นทาง 6 ขั้น

```
💡 Idea  →  ❓ Requirement  →  🎨 UX/UI
                                   ↓
🚀 Deploy  ←  ✅ Test  ←  🛠️ Build
```

| ขั้น | คำถามที่ตอบ |
|------|-------------|
| 💡 Idea | อยากได้อะไร |
| ❓ Requirement | ต้องมีอะไรบ้าง ใครใช้ |
| 🎨 UX/UI | หน้าตา/การใช้งานเป็นยังไง |
| 🛠️ Build | สั่ง AI สร้าง |
| ✅ Test | ลองใช้ + แก้ |
| 🚀 Deploy | ปล่อยให้คนใช้ |

---

## เส้นทางนี้ = โครงของทั้งคอร์ส

- โมดูล 05 → ❓ Requirement
- โมดูล 06 → 🎨 UX/UI
- โมดูล 07–11 → 🛠️ Build (frontend/backend/database)
- โมดูล 12 → ✅ Test + 🚀 Deploy

> 🗺️ ใช้เป็น "แผนที่" — รู้เสมอว่าตอนนี้อยู่ขั้นไหน

---

## เฉลย: นี่คือ SDLC

> "ที่เราจะทำกันทั้งคอร์สเนี่ย มืออาชีพเรียกว่า **SDLC** — วงจรการพัฒนาซอฟต์แวร์"

- และสไตล์ที่เราใช้ = ทำทีละนิด แล้ววนปรับกับ AI
- มืออาชีพเรียกสไตล์นี้ว่า **Agile**
- ซึ่งบังเอิญเป็นหัวใจของ **Vibe Coding** พอดี!

---

## หมายเหตุ: หลัง Deploy

เราจบที่ Deploy — แต่ของจริงยังต้อง "ดูแลต่อ" (มืออาชีพเรียก Maintain):
- 💰 ค่าใช้จ่ายเมื่อโต
- 🗄️ ข้อมูลเต็ม
- 🐛 แก้บั๊กที่เจอทีหลัง

> จะกล่าวถึงสั้น ๆ ตอนท้าย [[M12 สไลด์|โมดูล 12]] ไม่ใช่ขั้นแยก

---

## สรุป & ก้าวต่อไป

- การสร้างของจริงมี **ลำดับธรรมชาติ** 6 ขั้น
- เราจะ "เดินจริง" ไม่ใช่ท่องทฤษฎี
- สิ่งนี้คือ SDLC + Agile = หัวใจ Vibe Coding
