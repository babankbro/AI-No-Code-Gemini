# Database: ไต่ระดับ Google Sheets → Supabase (ไม่ใช่ใช้แทนกัน)

สอน database ที่ **ความลึกระดับ 2** (ตาราง/แถว/คอลัมน์/ความสัมพันธ์เบื้องต้น — ไม่แตะ SQL/key) ผ่านเครื่องมือ 2 ตัวแบบ *ไต่ระดับ* ไม่ใช่ทางเลือกเท่ากัน:

- **Google Sheets** = "database ตัวแรก" สำหรับ warm-up + concept (เพราะเป็นตู้เก็บเอกสาร/Excel ที่ทุกคนรู้จัก → สะพานเชื่อมแนวคิดดีสุด) และเป็น backend จริงได้สำหรับโปรเจกต์ A ที่ง่ายมาก
- **Supabase** = "ตัวจริงที่ใช้ปล่อยของ" ในเดโม (Postgres เบื้องหลัง, มีหน้าจอแก้ตารางเหมือนสเปรดชีต, รองรับความสัมพันธ์ + auth + deploy โดยไม่ต้องเขียน SQL)

เลือกไต่ระดับเพราะถ้าสอนปนกันแบบ "ใช้แทนกันได้" ผู้เรียนจะสับสนว่าทำไมมีสองตัว. **ผลที่ตามมา:** วัสดุเดโม Day 2 และตาราง "ข้อมูลที่ต้องเก็บ" จาก SRS จะผูกกับสคีมาแบบ Supabase — เปลี่ยนเครื่องมือภายหลังต้องรื้อหลายโมดูล.
