# แผนผังหลักสูตร — AI Vibe Coding สำหรับคนไม่มีพื้นฐาน

> สังเคราะห์จากการ grill ออกแบบ (ดูศัพท์กลางใน [CONTEXT.md](CONTEXT.md) และเหตุผลใน [docs/adr/](docs/adr/))

## ภาพรวม

- **กลุ่มเป้าหมาย:** A = เจ้าของไอเดีย/ธุรกิจ อยากสร้างของจริง (หลัก) · C = คนทั่วไปสายอยากเข้าใจยุค AI (รอง)
- **ปรัชญา:** Vibe Coding (AI เขียนโค้ดจริงให้) + สะพานเชื่อมแนวคิดสำหรับคนไม่มีศัพท์เทคนิค
- **แกนเล่าเรื่อง:** เส้นทาง 6 ขั้น 💡 Idea → ❓ Requirement → 🎨 UX/UI → 🛠️ Build → ✅ Test → 🚀 Deploy
- **เครื่องมือ:** Antigravity 2.0 + VS Code + Gemini · Database: Google Sheets → Supabase

## โมเดลการส่งมอบ — flipped classroom 3 วัน

| ช่วง | เวลา | เนื้อหา |
|------|------|---------|
| **Day 0 — เรียนด้วยตนเอง** | 1 วัน (async) | ดูดซับโมดูล *แนวคิด* แบบเต็ม (M01–M09 ส่วนทฤษฎี) + ติดตั้งเครื่องมือให้เสร็จ |
| **Day 1 + Day 2 เช้า** | 1.5 วัน | เวิร์กช็อปสด: ผู้สอนนำ **เดโมกลาง** + ผู้เรียน(A)ทำ **โปรเจกต์ตัวเอง** คู่ขนาน · กล่าวย่อ/ทวนแนวคิด |
| **Day 2 บ่าย** | 0.5 วัน | Build day: **โปรเจกต์ตัวเองล้วน ๆ** จนเสร็จ + deploy (lab + ผู้สอนช่วยรายคน) |

**หลักการจัดสรร:** ทุกแนวคิดเรียนเต็มที่ตอน self-study แต่ต้องถูกหยิบมา *ใช้จริง* ในเวิร์กช็อป — ไม่มีทฤษฎีไหนที่ไม่ได้แตะ
🏠 = self-study เต็ม · 🗣️ = กล่าวย่อ/ทวนตอนสด · 🛠️ = ลงมือทำสด

## เส้นเชื่อมที่ทำให้คอร์สเป็นเรื่องเดียวกัน (ไม่ใช่หัวข้อกระจัดกระจาย)

1. **Requirement → Database:** ตาราง "ข้อมูลที่ต้องเก็บ" จาก SRS (M05) = คอลัมน์ในตาราง Supabase (M09/M11)
2. **Requirement → Prompt:** ชุด 3 ชิ้นแมปตรงเข้า C-G-R-E-F (M05 ↔ M03)
3. **Requirement → Test:** เช็กลิสต์ "ต้องมี" จาก SRS = เช็กลิสต์ UAT (M05 ↔ M12) ปิด loop
4. **UX Flow/Wireframe → Build:** ภาพร่าง (M06) = input ของ prompt สร้าง frontend (M10)
5. **เดโมกลาง (Bean There จอง)** ร้อยทุกโมดูลเข้าด้วยกัน · กลุ่ม A ทำโปรเจกต์ตัวเองตามแพตเทิร์นเดียวกัน

## สิ่งที่ "จงใจไม่สอน" (ขอบเขตที่ขีดไว้)

- ❌ no-code tools ลากวาง (Bubble/Webflow) — คอร์สนี้คือ Vibe Coding
- ❌ SQL / primary-foreign key / normalization — ปล่อย AI จัดการ
- ❌ backend hands-on แยก — เป็นแนวคิดล้วน
- ❌ ทฤษฎีสี/ฟอนต์ — ปล่อย AI+Gemini
- ❌ Waterfall vs Agile เชิงทฤษฎี / unit test / automated test
- ❌ ขั้น Maintain เป็นโมดูลแยก — เป็นหมายเหตุท้าย Deploy
